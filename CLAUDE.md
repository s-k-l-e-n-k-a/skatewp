# Skate — Claude Instructions

## Workflow

- **No hacer commits ni push automáticamente.** Solo cuando el usuario lo pida explícitamente.
- Flujo: editar → probar en localhost → commit solo cuando el usuario lo pida → push solo cuando el usuario lo pida.

## Language

- **All backend UI text in English** — labels, headings, buttons, placeholders, notices.
- **All code comments and documentation in English** — inline comments, docblocks, README content.
- Exception: user-facing frontend text (if any) follows the site language.

## Stack

- **PHP** — lógica de backend, shortcodes, hooks de WordPress
- **Vanilla JS** — sin jQuery, sin frameworks; JS nativo para toda interactividad
- **SCSS** → compilado a CSS con `npm run build:css` (dart-sass + PostCSS/autoprefixer)
- **WordPress API** — usar siempre las APIs nativas de WP cuando existan:
  - `wp_options` para persistencia de datos
  - `wp_kses()` para sanitización de SVG/HTML
  - Variables CSS de theme.json (`var(--wp--preset--color--*)`, `var(--wp--preset--font-size--*)`) en lugar de valores hardcodeados
  - Hooks, nonces, capabilities para seguridad
  - `get_permalink()`, `home_url()`, etc. para URLs
- **Sin Composer, sin bundler de JS** — el tema no usa dependencias externas de PHP ni empaquetadores JS
- **Block system:** GreenLight blocks for layout/structure (sections, grids, cards, decorative elements). Core blocks (`wp:heading`, `wp:paragraph`, `wp:image`, `wp:list`) for editorial content. Core blocks inherit all typography from `theme.json` automatically — `theme.json` is the single source of truth for fonts and sizes.

## Admin Menu Order

The Skate admin pages must always appear in this order — both in the WP left sidebar and in the Skate topbar:

1. Settings
2. Identity
3. Design
4. Navigation
5. Sitemap
6. Reviews

Maintain this order whenever adding, removing, or renaming admin pages. The sidebar order is enforced via a late `admin_menu` hook (priority 999) in `inc/backend/general.php`. The topbar order is the `$nav` array in `skate_admin_topbar()`.

When adding a new page, also add its screen ID to the two `$skate_screens` allowlists in `general.php` (one in `skate_admin_topbar()`, one in the `in_admin_header` hook) — otherwise the topbar won't appear on that page.

## Admin Page Layout

Every Skate admin page must follow the same layout structure:

```php
<div class="wrap skate-identity-wrap">
    <h1>Page Title</h1>

    <?php if ( $saved ) : ?>
    <div class="notice notice-success is-dismissible"><p>Settings saved.</p></div>
    <?php endif; ?>

    <!-- optional tabs -->
    <!-- form / content -->
</div>
```

- Outer wrapper: `class="wrap skate-identity-wrap"`
- Title: plain `<h1>` with the page name (no icon, no suffix)
- Notices: standard WP `notice notice-success is-dismissible` immediately after the `<h1>`
- Call `skate_print_preset_assets()` before the wrapper to load shared admin CSS/JS

## CSS Class Prefix for User-Applied Classes

Classes intended to be added manually by users in the WordPress editor (block Advanced settings → Additional CSS class) use the prefix **`skate-`**. This makes them version-agnostic — they remain valid if the theme is versioned to Skate 2, Skate 3, etc.

Examples: `skate-submenu-featured-title`, `skate-no-shadow`

- `skate-` prefix serves double duty: internal theme classes (generated by PHP/SCSS) and user-facing utility classes documented in the UI.
